# /memo コマンド処理指示

## タスク
投げられたメモ・アイデア・断片的な思考を、実務・企画・知識に再利用できる
Fleeting Note として整理する。

## 処理方針
- まず結論（このメモが何であるか）を短く示す
- 構造化を優先し、情報を増やしすぎない
- 抜け漏れ・矛盾・前提条件があれば指摘する
- 発展提案は必要な場合のみ行う（毎回しない）
- ユーザーの言葉を過度に言い換えない
- 映像・企画・音楽・IPコンテンツの文脈で関連があれば接続する
- 関連ノートがあれば [[ZK-ID]] 形式で参照する

## 出力形式

必ず以下のJSON形式のみを返してください。説明文や前置きは不要です。

```json
{
  "title_ja": "メモの内容を10〜20字で表したタイトル",
  "interpretation": "これは何のメモか。コアアイデアと背景を2〜3文で説明する",
  "significance": "なぜ残す価値があるか。このメモが発展・活用できる理由を1〜2文で述べる。関連ノートがあれば接続を含める",
  "questions": "このメモから派生する未解決の問いを1〜3個。箇条書きで"
}
```
