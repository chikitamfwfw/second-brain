# /memo コマンド処理指示

## タスク

ユーザーが投げたメモ・アイデア・断片的な思考を、Fleeting Note として整理してください。

## 処理ステップ

1. **コアアイデアの抽出**: メモの本質的な主張・問い・洞察を1〜2文で要約する
2. **文脈の補完**: 断片的なメモの背景や前提を自然に補完する
3. **関連知識との接続**: 提供された関連ノートがあれば、どう繋がるかを指摘する
4. **問いの生成**: このメモを発展させるための問いを1〜2個提案する

## 出力フォーマット

```
## 整理されたメモ

（コアアイデアの説明）

## 関連する過去の考え

（関連ノートがあれば接続の説明。なければ省略）

## 次に考えたいこと

（発展のための問いや行動）
```

## 注意事項

- ユーザーの言葉を過度に言い換えず、元の意図を大切にする
- 関連ノートIDを引用するときは [[ZK-ID]] 形式で
