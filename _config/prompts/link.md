# /link コマンド処理指示

## タスク
記事・動画のコンテンツを、企画・実務・知識に再利用できる
Literature Note として整理する。

## 処理方針
- まずこのコンテンツが「何について・何を主張しているか」を一言で示す
- 要点は絞る（3点以内を基本とし、本当に重要なものだけ）
- 「自分ごと化」は毎回しない。映像・企画・音楽・IP文脈で接点が明確な場合のみ示す
- 示唆には批判的視点（前提の確認、反論可能性、注意点）を含める
- 関連ノートがあれば [[ZK-ID]] 形式で接点を示す
- アクション提案は実務・制作・判断に直結するものだけ

## 出力形式

必ず以下のJSON形式のみを返してください。説明文や前置きは不要です。

```json
{
  "summary": "内容の核心を2〜3文で。何を主張しているか・何がわかったか",
  "key_points": "重要ポイントを箇条書きで最大3点（Markdown形式: - 点1\n- 点2）",
  "details": "概要・要点に収まらなかった補足情報。なければ空文字",
  "insights": "この内容から読み取れる示唆・批判的観点・注意すべき前提を1〜2文",
  "personal_application": "映像・企画・音楽・IP文脈での転用可能性。関連性が薄い場合は空文字",
  "open_questions": "この内容から生まれる問いや深掘りポイントを1〜2個。なければ空文字"
}
```
