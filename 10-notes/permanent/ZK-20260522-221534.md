# 配信者コラボ企画は「自社IP×配信者／配信者同士／社内フロー整理」のどれかをまず切り分けるべき

「配信者コラボ」と一括りにすると論点が発散する。実務上は次の3つに分解できる。

1. 自社IP×配信者：IPを配信者に預けて訴求する案件型
2. 配信者同士：プロデューサーがマッチングや座組を組むコラボ型
3. 社内フロー整理：そもそもの企画・進行・契約フローを整える組織課題型

それぞれ目的（広告効果／作品性／オペレーション改善）も、KPIも、関係者も違うため、最初にどの軸の話かを確定させないと、相場感も告知設計も噛み合わなくなる。

# 配信者コラボはジャンルによって組み立て・権利処理・拡散導線が異なる

配信者コラボは「配信者」という括りで設計できない。ジャンルごとに文化・期待値・権利処理の前提が違うため、同じ企画書を当てはめると事故る。

- FPS・雑談系ストリーマー：ガチ感・即興性が価値。重い事前設計は嫌われやすい
- Vtuber：事務所単位のガイドライン・台本確認・事前告知が基本
- 歌い手／MVコラボ：楽曲権利処理とティザー設計が前提、プレミア公開とセットで動く

つまり「どのジャンルとやるか」を決めた瞬間に、許容される告知設計・契約フロー・拡散導線の選択肢がほぼ確定する。

# 配信コラボの告知タイミングは「事前／直前／サプライズ／事後」の4パターンに大別できる

告知設計は無数にあるように見えて、実務上は4類型に整理できる。

1. 事前告知型：数日〜数週間前にバナー付きで予告。期待値を積み上げる
2. 直前告知型：当日数時間前にSNSだけで告知。鮮度で引く
3. サプライズ型：配信開始時に初発表、または配信中に乱入。瞬間最大風速狙い
4. 事後告知型：終了後に切り抜き・クリップで拡散。ロングテール狙い

この4類型を共通言語にすることで、ジャンル別の相場感や、案件ごとのトレードオフを比較可能な形で議論できる。

# 告知タイミング設計は「期待値最大化」と「数字の取りこぼし防止」のトレードオフ

告知を早く出すほど認知は積み上がるが、本番同接のピーク感は薄れる。逆に遅らせるほど鮮度は出るが、リーチしきれない層が出てスポンサー満足度が下がる。

つまり告知タイミングは単なる運用Tipsではなく、「ピーク同接・話題化」と「総リーチ・広告効果」のどちらに寄せるかという設計判断である。KPIを先に決めずに告知タイミングだけ議論しても結論は出ない。

# 告知設計はジャンル文化に合っていないと、数字かスポンサー満足度のどちらかを失う

告知タイミングはジャンル文化との適合度で評価される。

- 企業案件・IPコラボ：事前告知でないと広告効果が測れず、スポンサーが納得しない
- FPS系ストリーマー：事前告知だと「作り込まれた感」が出てガチ感が損なわれ、サプライズ／直前告知の方がウケる
- Vtuber：事前告知が基本。例外は「ゲリラ凸待ち」など文化的に許容された形式のみ
- 歌ってみた・MVコラボ：事前ティザー＋プレミア公開設計が必須。サプライズは原則成立しない

ジャンル文化に逆らった告知設計は、本番同接が落ちるかスポンサーが納得しないかの二択を招く。

# 企画整理は抽象論より「現場で引っかかっている具体事象」から逆算したほうが早い

「配信者コラボの進め方を整理したい」のような抽象的な問いは、論点が広すぎて発散しやすい。整理を進めるには、現場で実際にモヤっている具体事象（例：事前告知で本番同接が落ちた、サプライズで数字は出たがスポンサーが納得しなかった）から逆算するほうが早い。

具体事象を起点にすると、関係する変数（ジャンル・KPI・関係者）が自動的に絞られ、抽象的なフレームワークも「その事象を説明できるか」という基準で取捨選択できるようになる。
